Introduction

You can find out more about RESTaurants API in this documentation. Some examples will be presented in a easy way. Users can use the documentation for further developing of the platform.


Why should i use the API?

Using the API will grant an easier access to locating different restaurants. Users can use the API to develop own ideas that includes location of restaurants. The data that is returned is a great resource for a unique platform that we think our users can develop. It is very easy to understand and use the API.


Target audience

We are targeting developers with a genuine interest of creating new ideas and platforms that solves problems. We do not have any other primary targets than coders.


Usage

Using the API is very simple. You can access our api with a simple URI ie: https://api.restaurantweb.com/rest/query?LOCATION results is going to be based on which location you are typing in. Location can be a city. The package will arrive in JSON.


Parameters

Using different parameters in the query is simple. You simply exchange LOCATION for your own location for example Malmo. The API will send you restaurants within 10km as standard, you can use radius paremeter to change the radius.

Returning package can arrive like this:

{"name":"M.E.A.T", "adress":"Streetwalk 11", "url":"http://www.meat.com", "phone":"030400200"};

query - query for locations
radius - set a radius for query
